## 附加说明

### 为什么最终分布需要重新计算百分比？
- 初始分布存储的是**比例**（0.10, 0.25等），直接乘以100即可
- 最终分布存储的是**人数**（1.10, 326.20等），需要先求和再计算百分比
- 代码第136-138行正确处理了这个转换：
  ```python
  initial_values = np.array([initial_dist[str(l)] * 100 for l in levels])
  total_final = sum(final_counts)
  final_values = np.array([(c / total_final) * 100 if total_final > 0 else 0 for c in final_counts])
  ```

### 为什么总和严格等于100%？
- 初始分布：人为设定的理论值，总和严格为1.0（100%）
- 最终分布：从10次运行的结果取平均，每次运行的总人数都是500，因此平均值总和也是500
- 转换为百分比时：(500 / 500) * 100 = 100.00%
